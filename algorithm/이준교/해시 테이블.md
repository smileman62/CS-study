# 해시 테이블

![해시 테이블](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdna%2Fb1zOw1%2FbtqL6HAW7jy%2FAAAAAAAAAAAAAAAAAAAAAEhT12GoZgO68ZR4bGwlxVddrbaFpG3e7rBVQLta8Yn8%2Fimg.png%3Fcredential%3DyqXZFxpELC7KVnFOS48ylbz2pIh7yKj8%26expires%3D1764514799%26allow_ip%3D%26allow_referer%3D%26signature%3DzHVSeAmwCfB%252B5AkaD8qS3BBvdy4%253D)

해시 테이블은 (Key, Value)로 데이터를 저장하는 자료구조 중 하나로 빠르게 데이터를 검색할 수 있는 자료구조이다.

해시 테이블이 빠른 검색 속도를 제공하는 이유는 내부적으로는 버킷을 사용해서 데이터를 저장하기 때문이다.

해시 테이블은 각각의 Key값에 해시 함수를 적용해 배열의 고유한 index를 생성하고 index를 활용해 값을 저장하거나 검색하게 된다. 실제 값이 저장되는 장소를 버킷 또는 슬롯이라고 한다.

해싱 구조로 데이터를 저장하면 Key값으로 데이터를 찾을 때 해시 함수를 한번만 수행하면 되기 때문에 매우 빠르게 데이터를 저장/삭제/조회할 수 있다. 해시 테이블의 평균 시간복잡도는 O(1)이다.

브루트 포스가 시간 초과에 빠지게 되는 문제들을 풀기 위해 해시 테이블을 사용한다. 브루트 포스의 경우에는 대부분 시간 복잡도가 O(n^2)이다.
